This guide will help you perform your first reverse geocoding query using Geocoder ID.

## Using the Go Library

<Steps>
  <Step title="Create a new Go file">
    Create a file called `main.go`:
    
    ```go
    package main
    
    import (
        "fmt"
        "log"
        
        geocoder "github.com/zsbahtiar/geocoder-id"
    )
    
    func main() {
        // Initialize geocoder (auto-downloads database if not found)
        gc, err := geocoder.NewDefault()
        if err != nil {
            log.Fatal(err)
        }
        defer gc.Close()
        
        // Reverse geocode coordinates (Jakarta)
        result := gc.ReverseGeocode(-6.2088, 106.8456)
        
        if result.Village != nil {
            fmt.Printf("Village: %s (%s)\n", result.Village.Name, result.Village.Code)
        }
        if result.District != nil {
            fmt.Printf("District: %s (%s)\n", result.District.Name, result.District.Code)
        }
        if result.Regency != nil {
            fmt.Printf("Regency: %s (%s)\n", result.Regency.Name, result.Regency.Code)
        }
        if result.Province != nil {
            fmt.Printf("Province: %s (%s)\n", result.Province.Name, result.Province.Code)
        }
    }
    ```
  </Step>
  
  <Step title="Run the program">
    ```bash
    go run main.go
    ```
    
    Expected output:
    ```
    Village: Pasar Manggis (31.74.02.1006)
    District: Setiabudi (31.74.02)
    Regency: Kota Administrasi Jakarta Selatan (31.74)
    Province: DKI Jakarta (31)
    ```
  </Step>
</Steps>

## Using the CLI

<Steps>
  <Step title="Basic reverse geocoding">
    Geocode a single coordinate pair:
    
    ```bash
    geocoder geocode --coords="-6.2088 106.8456"
    ```
    
    Output in table format:
    ```
    | province_code | province_name | regency_code | regency_name                      | district_code | district_name | village_code  | village_name  |
    |---------------|---------------|--------------|-----------------------------------|---------------|---------------|---------------|---------------|
    | 31            | DKI Jakarta   | 31.74        | Kota Administrasi Jakarta Selatan | 31.74.02      | Setiabudi     | 31.74.02.1006 | Pasar Manggis |
    ```
  </Step>
  
  <Step title="Get JSON output">
    For programmatic use, output as JSON:
    
    ```bash
    geocoder geocode --coords="-6.2088 106.8456" --output=json
    ```
    
    ```json
    [
      {
        "province": { "code": "31", "name": "DKI Jakarta" },
        "regency": { "code": "31.74", "name": "Kota Administrasi Jakarta Selatan" },
        "district": { "code": "31.74.02", "name": "Setiabudi" },
        "village": { "code": "31.74.02.1006", "name": "Pasar Manggis" }
      }
    ]
    ```
  </Step>
  
  <Step title="Geocode multiple coordinates">
    Process multiple coordinates at once using semicolon separation:
    
    ```bash
    geocoder geocode --coords="-6.2088 106.8456;-2.04963 110.18774"
    ```
    
    <Info>
      The second coordinate is in Central Kalimantan (Kasongan, Katingan Regency).
    </Info>
  </Step>
</Steps>

## Advanced Usage

### Query Specific Administrative Levels

You can limit results to a specific administrative level using the `--level` flag (CLI) or `ReverseGeocodeLevel()` method (Go library).

<CodeGroup>
```bash CLI - Province Level
geocoder geocode --coords="-6.2088 106.8456" --level=province
```

```bash CLI - District Level
geocoder geocode --coords="-6.2088 106.8456" --level=district
```

```go Go Library
// Query at different levels
result := gc.ReverseGeocodeLevel(-6.2088, 106.8456, geocoder.LevelProvince)
result := gc.ReverseGeocodeLevel(-6.2088, 106.8456, geocoder.LevelRegency)
result := gc.ReverseGeocodeLevel(-6.2088, 106.8456, geocoder.LevelDistrict)
result := gc.ReverseGeocodeLevel(-6.2088, 106.8456, geocoder.LevelVillage)
```
</CodeGroup>

Available levels:
- `province` - Province (Provinsi)
- `regency` - Regency/City (Kabupaten/Kota)
- `district` - District (Kecamatan)
- `village` - Village/Sub-district (Kelurahan/Desa) - *default*

### Batch Processing with Go

Process multiple coordinates efficiently:

```go
package main

import (
    "fmt"
    "log"
    
    geocoder "github.com/zsbahtiar/geocoder-id"
)

func main() {
    gc, err := geocoder.NewDefault()
    if err != nil {
        log.Fatal(err)
    }
    defer gc.Close()
    
    // Define coordinates to process
    coordinates := []geocoder.Coord{
        {Lat: -6.2088, Lon: 106.8456},  // Jakarta
        {Lat: -2.04963, Lon: 110.18774}, // Kalimantan
        {Lat: -7.7956, Lon: 110.3695},   // Yogyakarta
    }
    
    // Process each coordinate
    for _, coord := range coordinates {
        result := gc.ReverseGeocode(coord.Lat, coord.Lon)
        
        if result.Province != nil {
            fmt.Printf("%.4f, %.4f -> %s, %s\n", 
                coord.Lat, coord.Lon,
                result.Village.Name, result.Province.Name)
        }
    }
}
```

### Parsing Coordinate Strings

Use the built-in `ParseCoords()` function to parse coordinate strings:

```go
package main

import (
    "fmt"
    "log"
    
    geocoder "github.com/zsbahtiar/geocoder-id"
)

func main() {
    gc, err := geocoder.NewDefault()
    if err != nil {
        log.Fatal(err)
    }
    defer gc.Close()
    
    // Parse coordinate string (same format as CLI)
    coordStr := "-6.2088 106.8456;-2.04963 110.18774"
    coords := geocoder.ParseCoords(coordStr)
    
    for _, coord := range coords {
        result := gc.ReverseGeocode(coord.Lat, coord.Lon)
        if result.Province != nil {
            fmt.Printf("%s, %s\n", result.District.Name, result.Province.Name)
        }
    }
}
```

## Understanding the Result Structure

The `Result` struct contains pointers to each administrative level:

```go
type Result struct {
    Province *Location `json:"province,omitempty"`
    Regency  *Location `json:"regency,omitempty"`
    District *Location `json:"district,omitempty"`
    Village  *Location `json:"village,omitempty"`
}

type Location struct {
    Code string `json:"code"`
    Name string `json:"name"`
}
```

<Note>
  All fields are pointers and may be `nil` if the coordinate doesn't fall within any Indonesian administrative boundary.
</Note>

Always check for `nil` before accessing location data:

```go
result := gc.ReverseGeocode(lat, lon)

if result.Village != nil {
    fmt.Printf("Found: %s\n", result.Village.Name)
} else {
    fmt.Println("Coordinate not found in database")
}
```
